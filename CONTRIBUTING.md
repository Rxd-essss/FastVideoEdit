# Как внести вклад в FastVideoEdit

Спасибо за интерес к проекту! Ниже — краткий гид по dev-окружению, тестам и
принятым в проекте правилам.

## Dev-окружение (Windows)

Самый простой путь — лаунчер: он сам создаёт `.venv`, ставит зависимости и
запускает веб-редактор.

```powershell
.\run.ps1
```

Вручную (эквивалентно тому, что делает лаунчер):

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

Системные зависимости: `ffmpeg` / `ffprobe` (например, `winget install
Gyan.FFmpeg`) и, опционально, Ollama + `qwen3:8b` для LLM-функций.
Подробности — в [README](README.md#1-install-windows).

## Тесты

```powershell
.\.venv\Scripts\python.exe -m pytest -q
```

Сейчас в наборе 492 теста; перед отправкой изменений все должны быть зелёными.
Тесты не требуют GPU, ffmpeg или Ollama — внешние вызовы мокаются
(паттерны моков смотрите в существующих файлах `tests/`).

## Принципы кода

- **Логика пайплайна живёт в `vpipe/`** и переиспользуется и CLI
  (`pipeline.py`), и веб-редактором (`serve.py`). Не дублируйте логику в
  обёртках — добавляйте её в `vpipe/`.
- **Чистая логика покрывается тестами без ffmpeg/GPU**: детекторы,
  трансформации кат-листа, ремап субтитров и т.п. — на новые ветки логики
  добавляйте unit-тесты.
- Рендер и транскрипция — тонкие обёртки над ffmpeg / faster-whisper; их
  поведение тестируется через моки subprocess.

## Стиль

- Python: PEP8-ish — читаемость важнее буквы; осмысленные имена, докстринги
  для публичных функций.
- Строки интерфейса (веб-редактор, сообщения CLI) — на русском языке.
- Настройки и пороги — через `config.yaml`, не хардкодьте их в коде.
